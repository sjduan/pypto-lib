### 3.1 特定用例测试

**适用场景**：用户明确指定了具体的 shape 和 dtype（如："shape=[1,1024], dtype=float16"）

#### 测试范围

- ✅ **只测试用户明确指定的 shape 和 dtype**
- ✅ 至少需要 1 个测试用例（用户指定的配置）
- ✅ **可选**：测试用户指定 shape 的边界值（如极值、NaN 等）
- ❌ **不需要**测试其他 shape/dtype 组合
- ❌ **不需要**泛化到其他配置

#### 测试用例生成

```bash
# 示例：用户需求 shape=[1,1024], dtype=float16
python3 gen_golden.py "1,1024" -1 float16 test_specific_case

# 可选：边界值测试（仅限用户指定的 shape/dtype）
python3 gen_golden.py "1,1024" -1 float16 test_boundary_extreme
```

#### 准出条件（特定用例模式）

- [ ] 用户指定的 shape/dtype 测试通过
- [ ] 精度在容差范围内
- [ ] 测试报告已生成

**⚠️ 注意**：特定用例模式下，不需要满足 Level 0-4 的全量测试要求。

### 3.2 通用测试

**适用场景**：用户未明确指定 shape 和 dtype，或明确要求支持多种配置

#### 测试级别

| Level | 用例规模 | 目的 | 优先级 |
|-------|---------|------|--------|
| Level 0 | 8元素 | 验证基本逻辑 | P0 |
| Level 1 | 1K元素 | 验证正常功能 | P0 |
| Level 2 | 极值（0, -inf, +inf, NaN） | 验证边界条件 | P1 |
| Level 3 | 1M+元素 | 验证大数据量 | P2 |
| Level 4 | 10M+元素 | 性能测试 | P3 |

### 测试用例生成

**每个级别生成用例数量**：
- Level 0-2: 每个 shape/dtype 组合至少 2 个用例
- Level 3-4: 每个分支至少 1 个用例

```bash
# Level 0: 验证基本逻辑
python3 gen_golden.py "8" -1 float32 test_level0_1
python3 gen_golden.py "8" -1 float16 test_level0_2

# Level 1: 验证正常功能
python3 gen_golden.py "1024" -1 float32 test_level1_1

# Level 2: 验证边界条件
python3 gen_golden.py "1024" -1 float32 test_level2_extreme
```

### 精度验证

#### 精度标准

| 数据类型 | rtol | atol |
|---------|-----|------|
| FP32 | 1e-5 | 1e-6 |
| FP16 | 1e-3 | 1e-4 |
| 边界值 | 1e-3 | 1e-4 |

#### 验证方法

```python
import numpy as np

output = np.fromfile('output.bin', dtype=np.float32)
golden = np.fromfile('golden.bin', dtype=np.float32)

is_close = np.allclose(output, golden, rtol=rtol, atol=atol)
```

**如果精度不达标**：
1. 调用 `/ascendc-precision-debug` 查看精度调试指南
2. 分析误差原因
3. 修复后重新测试

## 阶段四：问题处理

### 遇到问题时的处理流程

当在开发过程中遇到问题时：

| 错误类型 | 处理方法 |
|---------|---------|
| **编译错误** | 定位错误行号，参考文档检查语法，对比官方示例 |
| **运行时错误** | 使用 `AscendC::printf` 定位问题，采用渐进式调试 |
| **精度错误** | 调用 `/ascendc-precision-debug` 进行系统化调试 |

### 禁止的行为

- ❌ 一遇到错误就全部重写
- ❌ 看到问题就简化代码
- ❌ 不分析原因就尝试其他方案

### 常见问题快速排查

| 问题 | 可能原因 | 解决方法 |
|-----|---------|---------|
| `ambiguous` 调用错误 | 函数定义顺序错误 | 先定义后调用，不用前向声明 |
| 编译失败 | 对齐问题 | 检查 32/64/512 字节对齐 |
| 精度错误 | 数据类型不匹配 | 确认 API 支持的类型 |
| 全局聚合结果错误 | 分段独立计算 | 设计多遍遍历方案 |
