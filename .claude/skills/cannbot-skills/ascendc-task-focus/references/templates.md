## 格式建议

### 步骤命名

```markdown
- [ ] 动作 + 对象：说明
```

示例：
```markdown
- [ ] 创建算子目录：在 ops/ 下创建子目录
- [ ] 编写 kernel 代码：实现核心计算逻辑
- [ ] 编译测试：验证代码无语法错误
```

### 优先级标记

```markdown
- [ ] 🔥 高优先级：关键路径
- [ ] 中优先级：正常步骤
- [ ] 低优先级：可选优化
```

### 依赖关系

```markdown
- [ ] 步骤 A
- [ ] 步骤 B (依赖: A)
- [ ] 步骤 C (依赖: A, B)
```

### 预估时间

```markdown
- [ ] 步骤 A [15分钟]
- [ ] 步骤 B [30分钟]
- [ ] 步骤 C [1小时]
```

### 状态标记

```markdown
- [ ] 待办
- [x] 已完成
- [~] 进行中
- [!] 阻塞
- [?] 待确认
```
