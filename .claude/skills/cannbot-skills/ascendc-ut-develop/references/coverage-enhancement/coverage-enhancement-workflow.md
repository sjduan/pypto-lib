# 覆盖率增强工作流程

> 本文档从 SKILL.md 抽离，详细描述 UT 已存在时的覆盖率提升流程。

---

## 适用场景

**触发条件**：`tests/ut/` 目录下已存在 `test_*.cpp` 测试文件

**目标**：将现有覆盖率提升至 90%+

---

## Phase CE-1: Interview 模式

收集算子基本信息：

| 信息项 | 说明 |
|--------|------|
| 层级 | op_host / op_api / op_kernel |
| 算子名称 | 具体算子名称（注意：不要带v2/v3后缀，除非明确需要） |
| 输出路径 | `{OUTPUT_DIR}/{OPERATOR_NAME}` |

### 输出路径结构

```
{OUTPUT_DIR}/{OPERATOR_NAME}/
├── log/           # 编译测试日志
├── cov_result/    # 覆盖率信息
├── code/          # 修改文件的副本（最终测例）
└── report/        # 报告
```

**注意**：最终输出到 code/ 目录的代码必须是测试全部通过的测例，使覆盖率最高。

---

## Phase CE-2: 获取初始覆盖率

调用子智能体获取初始覆盖率基线。

**调用方式**：
```
Task 工具 → general-purpose 子智能体
任务：获取初始UT覆盖率
```

**必须传递给子智能体的信息**：
1. 编译指令（包含 --cov 参数）
2. SOC版本列表（查看算子文档确认支持的SOC）
3. 日志保存路径

**编译指令模板**：
```bash
cd {PROJECT_DIR} && bash build.sh -u --ophost --ops='{operator_name}' --soc='{soc_version}' --cov 2>&1 | tee {outputdir}/{op_name}/log/round_0.log
```

详细指令参见 [基线覆盖率获取智能体](baseline-coverage-agent.md)

---

## Phase CE-3: 分析与实现测试用例

获得初始覆盖率后，分析未覆盖代码区域，调用子智能体实现测试用例。

**调用方式**：
```
Task 工具 → general-purpose 子智能体
任务：实现测试用例以覆盖特定代码分支
```

**必须传递给子智能体的信息**：
1. 编译指令
2. SOC版本列表
3. 未覆盖代码分析

**关键要求**：
- 每轮迭代新增的测例必须全部通过（`[  FAILED  ]` 标识表示测试不通过）
- 如无法修复，可删除不通过的测例，确保剩余测例通过
- 迭代中覆盖率不应下降

详细指令参见 [测试用例实现智能体](test-implementation-agent.md)

### 迭代优化流程

1. 首次调用测试用例实现智能体
2. 分析反馈结果：
   - 成功覆盖 → 继续尝试其他分支
   - 失败 → 分析原因并重新设计
3. 重复调用直到无法继续提升
4. 若仍有未测试通过的测例，继续调度修复
5. 若还有未尝试的文件，继续尝试

### 目标

- 总覆盖率达到 90%+（覆盖率增强模式目标，高于 UT 生成模式的 80% 基准）
- 所有文件覆盖率达到 85%+

---

## Phase CE-4: 持续迭代优化

每次调用子智能体后，评估覆盖率变化：

```markdown
# 覆盖率评估报告

## 调用前覆盖率
- 总覆盖行数：[X]
- 未覆盖行数：[Y]

## 调用后覆盖率
- 新增覆盖行数：[Z]
- 覆盖率提升：[N]%

## 未覆盖代码状态
- 可覆盖代码：[数量]
- 无法覆盖代码：[数量]
- 原因分析：[具体说明]
```

继续迭代直到覆盖率达到实际能提升到的最优状态。

---

## Phase CE-5: 生成最终报告

### 可覆盖代码报告格式

```markdown
#### 文件：xxx_tiling.cpp:行号范围
**代码片段**：[复制关键代码]
**进入条件**：
- 参数1：[值/范围]
- 参数2：[值/范围]
- 场景要求：[描述]
**说明**：[为什么需要这些条件]
```

### 无法覆盖代码报告格式

```markdown
#### 文件：xxx.cpp:行号
**类型**：[死代码/不参与UT/条件不可能满足]
**原因**：[具体说明]
```

---

## Phase CE-6: 输出进入条件清单

```markdown
# 进入条件清单

## 可覆盖代码

| 代码位置 | 进入条件 | 所需参数 |
|---------|---------|---------|
| xxx.cpp:123 | dtype=INT8, 有quantScale | aType=FP16, bType=INT8, 提供antiQuantScale |
| xxx.cpp:456 | shape满足特定对齐 | M=16, K=64 |

## 无法覆盖代码

| 代码位置 | 类型 | 原因 |
|---------|------|------|
| xxx.cpp:789 | 死代码 | useMmOutputAsX1Input硬编码为true |
| fallback.cpp:1-100 | 不参与UT | fallback实现，不在ophost范围 |
```
