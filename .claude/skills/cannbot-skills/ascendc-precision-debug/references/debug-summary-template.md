## 示例：完整填写案例

```markdown
# 精度调试总结

## 本次调试信息
- **算子名称**：Sinh
- **调试日期**：2025-01-15
- **问题症状**：FP16 精度下，输入 x > 5 时结果误差超过 1e-2

## 1. 文档描述不清晰的地方
- **具体描述**：Exp API 的返回范围说明不明确
  - 位置：`asc-devkit/docs/api/context/Exp.md`
  - 问题：未说明 FP16 下 Exp 的溢出阈值
  - 改进建议：补充不同 dtype 下的有效输入范围表格

## 2. 需要改进的地方

### Skill 改进
- **ascendc-precision-debug**：快速调试方法的触发条件不够明确
  - 问题：SKILL.md 中没有明确说明何时使用哪种快速方法，导致 Agent 选择困难
  - 建议改进：增加"调试方法选择决策树"，根据误差类型快速选择方法
  - 影响程度：中

### 其他
- **示例代码**：缺少处理数值溢出的完整示例
  - 建议增加：展示如何使用 max-log-sum-exp 技巧避免溢出

## 3. 给 Agent 造成困扰的点
- **具体困扰**：不知道如何判断 Exp 是否溢出
  - 困扰描述：花了 3 次尝试才发现是 exp(x) 直接溢出导致 NaN
  - 改进建议：在 SKILL.md 的"常见陷阱"中增加"exp/log 溢出"条目

## 4. 调试过程记录
- **尝试次数**：4 次
- **使用方法**：Printf 调试 + 常见陷阱排查
- **最终方案**：先减最大值再计算 exp，避免溢出
- **验证结果**：FP16 max_error 从 1e-2 降至 5e-4，通过所有测试
```
