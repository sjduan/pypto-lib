## 版本兼容性注意事项

### 1. CANN 版本差异

**当前情况**：
- 项目环境：CANN 8.5.0
- asc-devkit：版本 9.0.0

**影响**：
- 部分新 API 可能需要 CANN 9.0.0 支持
- 建议查阅 API 文档确认版本要求
- 使用新 API 前先验证当前环境兼容性

### 2. 数据类型支持

| 类型 | 说明 |
|-----|------|
| `float` | 32 位浮点 |
| `float16` / `half` | 16 位浮点 |
| `bfloat16` | BFloat16 格式 |
| `int8` / `uint8` | 8 位整数 |
| `int16` / `uint16` | 16 位整数 |
| `int32` / `uint32` | 32 位整数 |
| `int64` / `uint64` | 64 位整数 |

### 3. 对齐要求

| 操作类型 | 对齐要求 |
|---------|---------|
| 大多数矢量操作 | 32 字节 |
| DataCopy | **512 字节** |
| 某些特殊 API | 64/128 字节 |

**查阅方法**：查看对应 API 文档的 **Restriction** 章节

## 使用建议

1. **查阅 API 文档时**：
   - 优先查看 **Restriction** 章节
   - 确认 API 支持的 NPU 架构
   - 验证对齐要求

2. **使用新 API 前**：
   - 检查 CANN 版本要求
   - 在当前环境测试验证
   - 备选兼容方案

3. **开发新算子时**：
   - 确认目标服务器类型
   - 选择兼容的 API
   - 注意数据类型限制

4. **遇到兼容性问题时**：
   - 查阅 `examples/10_compatibility_cases/` 示例
   - 参考 asc-devkit 的兼容性处理方法
   - 降级使用旧版本 API
